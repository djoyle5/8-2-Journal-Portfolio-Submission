# 8-2-Journal-Portfolio-Submission
For my “Draw It or Lose It” design doc, The Gaming Room wanted a web-based version of their drawing/guessing game where multiple teams can play at the same time. The main focus was making sure it could handle lots of users, run smoothly, and work across different devices.

I think I did a good job organizing the design so it’s easy to follow and not overly technical. I also connected the requirements to actual design choices instead of just listing them out.

The design doc will help a lot with impoving the coding because it forced me to plan everything first, like how the system should be structured and how parts would work together before jumping into the code.

If I could change one thing, I’d improve the scalability section and try to learn more about cloud networks to make a more confident definitive choice.

For user needs, I focused on making sure the game is smooth and easy to use in real time. That’s important because if the user experience is bad, the whole service kind of fails even if the code works, no one would want to use it.

I just started with the requirements, broke everything down, and planned it step by step. This is a solid way for me to work through larger projects in the future. I’d spend a little more time on planning and diagrams to make coding even easier.
